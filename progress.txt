## Codebase Patterns
- Build: `npm run build` (vite), `npm run typecheck` (tsc --noEmit), `npm run dev` (vite dev)
- @crxjs/vite-plugin@beta (v2) handles manifest rewriting and service worker loader generation
- manifest.json at project root is the source; crxjs rewrites paths in dist/manifest.json
- Icons at public/icons/{16,48,128}.png — crxjs copies them into dist/
- TypeScript: ES2022 target, strict, ESNext modules, bundler resolution
- No UI framework — vanilla TS/HTML/CSS
- Shared types in `src/background/types.ts` — import from there for all interfaces
- AXNode response from Accessibility.getFullAXTree wraps role/name/value in {type, value} objects
- Only increase indentation depth when a node receives a ref assignment
- ActionExecutor.execute() is a switch dispatcher — add new action cases as they're implemented
- CDP response types need local interfaces (BoxModelResponse, ResolveNodeResponse, etc.) since @types/chrome doesn't type sendCommand results
